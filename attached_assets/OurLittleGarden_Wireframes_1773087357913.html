4
Drawing Studio — Detail
Close-up of every interaction inside the drawing canvas panel.
Drawing Flow
1
User selects brush type (round / petal / glitter) from tool row
2
Picks brush size via mini slider (XS → XL, displayed as live dot)
3
Picks colour from 24-swatch grid. Eyedropper tool samples from canvas.
4
Draws on canvas. Catmull-Rom smoothing applied to mouse/touch points for fluid curves.
5
Undo (⌘Z) or Redo (⌘Y) — stack of up to 30 canvas ImageData snapshots.
6
Happy with result → clicks "Save Flower 🌸". Canvas must not be blank.
7
Sparkle burst animation plays. Flower appears in library grid instantly.
Save Flower Flow
1
canvas.toBlob('image/webp', 0.85) → Blob object
2
uploadBytes(storageRef, blob) to Firebase Storage at /flowers/{uuid}.webp
3
getDownloadURL(storageRef) → permanent public URL
4
set() in Realtime DB: {imageUrl, drawnBy, inGarden:false, savedAt}
5
incrementHealth(+5). Trigger sparkle animation. Clear canvas for next drawing.
6
Both users' onValue listener fires → thumbnail appears in both library grids instantly

A
Brush Types
Round: standard ctx.arc stroke. Petal: stamp a small SVG petal shape at each point. Glitter: scatter 5–8 tiny random dots ±10px around cursor position per frame.
B
Brush Size Slider
Custom range input styled with CSS. Live preview dots (3 sizes shown) update in real time. Range: 2px (XS) → 30px (XL). Default: 8px (M).
C
Stroke Smoothing
Store raw points in array. On mousemove, use ctx.quadraticCurveTo() between midpoints of consecutive point pairs. Prevents jagged lines on fast strokes.
D
Optional Name
Caveat font input below canvas. If filled, stored in Firebase as flower.name. Shows as a tooltip on hover in the library grid and as a label in the garden.
E
Rainbow Swatch
Last swatch opens a mini colour picker (native input type="color") so users can pick any custom colour not in the preset palette.

A
CSS Filter System
A single CSS custom property --health is set on the .garden-flowers wrapper. Each placed flower inherits filter: saturate(calc(var(--health) * 1.2%)) automatically.
B
Wilt Transform
Below 50%: CSS transition on each flower adds rotate(6deg) translateY(4px). Below 20%: increases to rotate(12deg) translateY(8px). Smooth 3s ease transition.
C
Petal Fall (< 40%)
CSS keyframe animation creates 6–8 tiny petal divs at random horizontal positions. They fall with slight horizontal drift and fade out over 4–7s, then loop.
D
Recovery Bloom
When health increases (letter sent), Framer Motion runs on each flower: scale(0.9→1.05→1) + rotate(current→0). Spring physics make it feel organic.
E
Gentle Nudge Toast
When health drops below 30%, a soft non-intrusive message appears at the bottom of the garden: "The garden misses you 🥀 Send a letter!" — not a scary alert.

Notification Permission Flow
1
User sends first letter → app requests Notification.requestPermission()
2
If granted: register Service Worker for background notifications
3
Firebase onValue → if app backgrounded → new Notification(title, body)
4
Click notification → window.focus() + navigate to /garden/{code}?open=mailbox
A
Mailbox Badge Logic
unreadCount = Object.values(mailbox).filter(l => l.to === userId && !l.readAt).length. Derived in mailStore. Badge re-renders instantly on Firebase update.
B
Toast Notifications
Use react-hot-toast or a custom div. Slide in from top-right. Auto-dismiss after 4s. Soft, gentle styling — not aggressive red alerts. Position: top-right, not fullscreen.
C
Friend Joins
onValue on /users — if a second user appears that wasn't there before, trigger a toast "🦊 Zara joined your garden!" + both avatar icons appear in top bar.
D
No Emails / SMS
Intentional design choice: the app uses only browser notifications. No external services needed. This keeps the MVP simple and avoids email infrastructure costs.
